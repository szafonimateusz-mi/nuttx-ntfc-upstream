============
Test Reports
============

Overview
========

NTFC generates comprehensive test reports that track test results, execution
times, and pass/fail statistics. Reports are organized per test module with
an aggregated summary across all modules.

Result Directory Layout
=======================

After running tests the result directory has the following structure::

  <session_dir>/
      pytest.debug.log                          # full pytest debug log
      <product>/<core>/<testname>.console.txt   # raw console output per test
      <product>/<core>/<testname>.device.txt    # device events per test
      report/
          001_<module_name>.xml                 # per-module JUnit XML
          001_<module_name>.html                # per-module HTML report
          002_<module_name>.xml
          002_<module_name>.html
          ...
          result_summary.txt                    # aggregated text summary
          result_summary.html                   # aggregated HTML summary

Report Files
============

Debug Log (located in ``<session_dir>/``):

- ``pytest.debug.log`` - Full pytest debug-level log for the session

Raw Pytest Reports (generated by pytest, located in ``<session_dir>/``):

- ``report.xml`` - Combined JUnit XML from all tests; automatically split into
  per-module files and then removed
- ``report.json`` - JSON format report (only if ``--json`` option used and
  pytest-json plugin available)
- ``report.html`` - Detailed pytest-html report; removed after splitting

Per-Module Reports (generated by NTFC, located in ``<session_dir>/report/``):

- ``001_<module_name>.xml`` - JUnit XML format for the module
- ``001_<module_name>.html`` - Human-readable HTML report for the module

Summary Reports (generated by NTFC, located in ``<session_dir>/report/``):

- ``result_summary.txt`` - Text summary with statistics table
- ``result_summary.html`` - HTML summary with links to module reports

Report Generation
=================

Reports are automatically generated at the end of test execution:

1. Pytest generates ``report.xml`` in the session directory
2. ``report.xml`` is split into per-module files under ``report/``
   (e.g., ``report/001_test_module1.xml``)
3. Each module XML generates a corresponding HTML report in ``report/``
4. ``report.xml`` and ``report.html`` are removed from the session directory
5. The final summary aggregates all module results into
   ``report/result_summary.txt`` and ``report/result_summary.html``

Report Contents
===============

Each module report includes:

- Test Statistics: Pass/fail/error/skipped counts
- Execution Time: Total time for all tests in the module
- Pass Rate: Percentage of passing tests
- Test Details: Individual test results with status and error messages

The summary report provides:

- Module List: All modules tested with their individual statistics
- Overall Statistics: Total tests, passes, failures, errors, skipped
- Total Execution Time: Sum of all test execution times
- Hyperlinks: Direct links to individual module reports
